## 🏗️ Building Panovia

> **Panovia** is an AI-native project-memory and knowledge layer for **AEC** (Architecture, Engineering & Construction) teams — an agentic assistant that reads a project's real source-of-truth and answers with evidence instead of guesses.

As **CTO at [Attimo Technologies](https://www.linkedin.com/in/syed-osama-ali-shah-a74a23215)** (UK-incorporated, engineering out of Ankara), I lead the technical direction and the build:

- 🧩 **Native Revit / IFC ingestion** via ODA BimRv — parsing BIM models at the source, not from flattened exports. *(the moat)*
- 🕸️ **Hybrid retrieval** — a **Neo4j** project graph + vector search (GraphRAG) over a **Postgres** evidence ledger, so every answer is traceable.
- ⚡ **Latency engineered down ~49s → ~2.5s**, with a region pipeline that cut VLM calls per drawing page by **~98%**.
- 🤖 **Multi-model routing** through OpenRouter; **FastAPI** backend, **Next.js** frontend on **GCP**.
- 🎓 Backed by the **UCL Hatchery** at BaseKX.
